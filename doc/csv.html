Transformenator CSV Transform

The ExtractCSV utility function takes a file and divides it up into fixed-length records with fixed-length fields that you specify and creates a comma-separated values (CSV) file in the process.

The table below defines the keywords and values used to define an interpretation of a file as records. All numerical values are given in hexadecimal.
Keyword Explanation Example
; Comment, will be ignored ; This text will not be considered
FirstRec Offset (in hex bytes) to the first record to be considered FIRSTREC=0E00
NextRec Offset (in hex bytes) to the next record NEXTREC=100 ; One record every 0x100 bytes
SelectIf (Optional) criteria that must be met to consider a record for output; specification first lists the offset within the record, followed by a comma, followed by one or more hex bytes that must match. Multiple SelectIf clauses may be used to produce logical OR conditions. SELECTIF=00,40 ; First byte must be 0x40 to be valid

The table below defines export instructions for a single field. Multiples of these specifications make up a full record to export. One each of Name, Origin, Length, and Interp must be present. All numerical values are given in hexadecimal.
Keyword Explanation Example
; Comment, will be ignored ; This text will not be considered
Name Field name; will be exported on header row of output CSV file NAME="Field 1"
Origin Offset within the record to this field's data ORIGIN=01
Length Length of this field's data LENGTH=0a
Interp Interpretation of field data on export - one of:
"ASCII" - ASCII text
"EBCDIC" - EBCDIC text
"HEX" - Hexadecimal notation		 INTERP="EBCDIC"
